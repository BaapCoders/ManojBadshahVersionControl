## Security Considerations

1. **S3 Bucket Access:**
   - Public read for preview URLs
   - OR: Use presigned URLs (expires in 1 hour)

2. **AWS Credentials:**
   - Stored in `.env` (not committed to git)
   - IAM policy: `s3:PutObject`, `s3:GetObject` only

3. **Canvas Serialization:**
   - No sensitive data in JSON
   - Images not serialized (only metadata)

4. **API Authentication:**
   - Currently open (no auth)
   - TODO: Add JWT/session auth for production

---

## Performance Optimization (Future)

1. **Caching:**
   - Cache serialized JSON in Redis
   - Cache PNG thumbnails in CDN

2. **Compression:**
   - Gzip JSON before sending to Python
   - Compress PNG (80% quality) before S3 upload

3. **Async Processing:**
   - Queue PNG generation (RabbitMQ/Bull)
   - Return version immediately, generate PNG in background

4. **Thumbnails:**
   - Generate 800x800px thumbnail instead of full 1080x1080
   - Reduces S3 storage by ~60%

---

This architecture provides a scalable, cloud-native version control system with visual previews!
